# 10 - 首页 (index.html)：文章列表

> 视频讲解稿 · 对应文件：[frontend/index.html](../../frontend/index.html)

## 开场

首页是整个博客的门面，也是用户看到的第一页。它展示所有已发布的文章，每篇显示标题、作者、发布时间和 150 字摘要。页面结构分为三部分：导航栏、内容区、脚本。

---

## HTML 结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>博客首页</title>
  <link rel="stylesheet" href="/css/style.css">
</head>
```

- **`<!DOCTYPE html>`** — 声明文档类型为 HTML5
- **`lang="zh-CN"`** — 声明页面语言为简体中文，帮助搜索引擎和屏幕阅读器
- **`viewport` meta** — 移动端适配。`width=device-width` 让页面宽度等于设备宽度，`initial-scale=1.0` 初始缩放比例为 1

---

### 导航栏

```html
<nav class="navbar">
  <a href="/" class="logo">📝 简易博客</a>
  <div class="nav-links" id="navLinks"></div>
</nav>
```

- **`<nav>`** — HTML5 语义化标签，表示导航区域
- **`id="navLinks"`** — 这个 div 的内容由 `updateNavbar()` 动态填充，根据登录状态显示不同内容

**导航栏在页面加载后才填充**，因为需要读取 localStorage 判断登录状态。

---

### 内容容器

```html
<div class="container">
  <div class="page-header">
    <h1>最新文章</h1>
  </div>
  <div id="alertBox" class="alert"></div>
  <div id="postList"></div>
</div>
```

- **`id="alertBox"`** — 消息提示区，初始隐藏（`display: none`）
- **`id="postList"`** — 文章列表容器，由 JS 动态填充

### 列表渲染

```javascript
    const html = posts.map(p => `
      <div class="post-item">
        <div class="post-title">
          <a href="/detail.html?id=${p.id}">${escapeHtml(p.title)}</a>
        </div>
        <div class="post-meta">
          👤 ${escapeHtml(p.author)} · 🕐 ${formatDate(p.created_at)}
        </div>
        <div class="post-summary">${escapeHtml(p.summary || '')}</div>
      </div>
    `).join('');

    container.innerHTML = `<ul class="post-list">${html}</ul>`;
```

#### `posts.map(fn).join('')` 模式

```javascript
// map 把每个元素转成 HTML 字符串，得到一个字符串数组
['<div>...</div>', '<div>...</div>', '<div>...</div>']

// join('') 把数组拼接成一个长字符串
'<div>...</div><div>...</div><div>...</div>'
```

#### 数据到 UI 的映射

```
后端返回的 JSON 对象              渲染出的 HTML
─────────────────────            ──────────────
p.id        = 1      →  /detail.html?id=1  (链接 URL)
p.title     = "Hello" →  文章标题 (可点击)
p.author    = "alice" →  👤 alice
p.created_at= "..."   →  🕐 3 小时前
p.summary   = "..."   →  摘要文本
```

#### `escapeHtml()` 的重要性

所有来自后端的用户数据（title, author, summary）都经过 `escapeHtml()` 处理，防止 XSS 攻击。

### 页面初始化

```javascript
document.addEventListener('DOMContentLoaded', loadPosts);
```

页面加载完成后自动调用 `loadPosts()`。注意 `main.js` 中也监听了 `DOMContentLoaded` 来更新导航栏——两个监听器互不干扰，都正常执行。

## 数据流总结

```
页面加载
  ├─ DOMContentLoaded → updateNavbar()  (来自 main.js)
  └─ DOMContentLoaded → loadPosts()     (本页脚本)
       │
       ├─ get('/posts')
       │    └─ fetch('/api/posts')
       │         └─ Express → db.prepare(SELECT...JOIN...).all()
       │              └─ 返回 [{id, title, summary, author, created_at}, ...]
       │
       ├─ posts.length === 0 → 显示空状态
       │
       └─ posts.map() → 渲染文章卡片列表
```

---

## 总结

首页代码量不大，但展现了前端开发的几个核心模式：
1. **数据驱动 UI** — 拿到数据 → 遍历生成 HTML → 注入 DOM
2. **空状态处理** — 有意为之，而非"忘了处理"
3. **条件渲染** — 根据登录状态显示不同内容
4. **防御性编程** — escapeHtml 无处不在
