## 第一部分：文章详情页 [detail.html]

### 页面职责

1. 展示文章完整内容（标题、作者、时间、正文）
2. 判断当前用户是否是作者
3. 如果是作者——显示编辑和删除按钮

---

### HTML 结构

```html
<div class="container">
  <div id="alertBox" class="alert"></div>
  <div id="postDetail"></div>     <!-- 全部由 JS 动态渲染 -->
</div>
```

这个页面几乎全部由 JS 动态渲染——服务器返回的 HTML 只是一个空壳。

---

### JS：加载详情

```javascript
async function loadDetail() {
  const params = new URLSearchParams(window.location.search);
  const id = params.get('id');
  const container = document.getElementById('postDetail');

  if (!id) {
    container.innerHTML = '<div class="empty-state"><p>文章 ID 无效</p></div>';
    return;
  }
```

跟 create.html 一样读取 URL 参数 `?id=`。如果没有 id，显示错误提示。

---

### 关键：作者身份判断

```javascript
  const post = await get(`/posts/${id}`);
  const user = getCurrentUser();                   // 从 localStorage 读取
  const isOwner = user && user.id === post.user_id; // 比对 ID
```

**`isOwner` 的逻辑**：

```javascript
user          post.user_id    isOwner
─────         ────────────    ───────
null          1               false    (未登录)
{id: 2}       1               false    (登录了但不是作者)
{id: 1}       1               true     (是作者)
```

- 用户在未登录状态下也能看文章详情
- 只有登录 + 是作者才能看到操作按钮

---

### 条件渲染操作按钮

```javascript
  const actionsHtml = isOwner ? `
    <div class="post-actions">
      <a href="/create.html?id=${post.id}" class="btn btn-secondary">✏️ 编辑</a>
      <button class="btn btn-danger" onclick="deletePost(${post.id})">🗑 删除</button>
    </div>
  ` : '';
```

**三元表达式 `condition ? truthy : falsy`**：

- 是作者 → 显示编辑/删除按钮
- 不是作者 → 空字符串（不显示按钮）

---

### 更新时间显示

```javascript
<div class="post-meta">
  👤 ${escapeHtml(post.author)} · 🕐 ${formatDate(post.created_at)}
  ${post.updated_at !== post.created_at
    ? ` · 更新于 ${formatDate(post.updated_at)}`
    : ''}
</div>
```

只有当 `updated_at` 和 `created_at` 不同时，才显示"更新于"标签。

```
首次发布：  👤 alice · 🕐 3 小时前
编辑过：    👤 alice · 🕐 3 小时前 · 更新于 1 小时前
```

---

### 删除功能

```javascript
async function deletePost(id) {
  if (!confirm('确定要删除这篇文章吗？此操作不可撤销。')) return;

  try {
    await del(`/posts/${id}`);
    alert('删除成功');
    window.location.href = '/';
  } catch (err) {
    showAlert('alertBox', err.message, 'error');
  }
}
```

- **`confirm()`** — 浏览器原生确认对话框，返回 `true`（确定）或 `false`（取消）
- 确认 → `del()` 发送 DELETE 请求 → 删除成功 → 跳回首页
- 取消 → `return` 什么都不做

## 第二部分：我的文章页 [my.html]

### 页面职责

展示当前用户发布的所有文章，每篇文章带"编辑"和"删除"按钮。这是博客的"管理中心"。

---

### 登录守卫

```javascript
if (!isLoggedIn()) {
  window.location.href = '/login.html';
}
```

和 create.html 一样，顶层立即执行的登录检查。

---

### 加载我的文章

```javascript
async function loadMyPosts() {
  const container = document.getElementById('postList');

  try {
    const posts = await get('/posts/my');  // 需要 Token，返回当前用户的文章
```

- **`get('/posts/my')`** → `api.js` 自动附加 Authorization header
- 后端路由 `/posts/my` 经过 `authMiddleware` 鉴权 → `WHERE p.user_id = ?` 仅返回当前用户的文章

---

### 列表项：区别于首页

```javascript
    const html = posts.map(p => `
      <div class="post-item">
        <div class="post-title">
          <a href="/detail.html?id=${p.id}">${escapeHtml(p.title)}</a>
        </div>
        <div class="post-meta">
          🕐 ${formatDate(p.created_at)}
          ${p.updated_at !== p.created_at ? ` · 更新于 ${formatDate(p.updated_at)}` : ''}
        </div>
        <div class="post-actions">                                 <!-- ← 区别在这里 -->
          <a href="/create.html?id=${p.id}" class="btn btn-secondary btn-sm">编辑</a>
          <button class="btn btn-danger btn-sm" onclick="deleteMyPost(${p.id})">删除</button>
        </div>
      </div>
    `).join('');
```

和首页列表的差异：
- 首页：显示摘要（150 字），无操作按钮
- 我的文章：显示操作按钮（编辑/删除），不显示摘要

---

### 原地刷新删除

```javascript
async function deleteMyPost(id) {
  if (!confirm('确定要删除这篇文章吗？此操作不可撤销。')) return;

  try {
    await del(`/posts/${id}`);
    loadMyPosts();  // 重新加载列表，不跳转页面
  } catch (err) {
    showAlert('alertBox', err.message, 'error');
  }
}
```

和详情页的删除不同——这里删除后不跳转首页，而是**原地刷新列表**。这样用户可以连续删除多篇文章，体验更好。

```javascript
// 详情页删除 → 跳转首页（文章没了，详情页已无意义）
window.location.href = '/';

// 我的文章删除 → 重新加载列表（留在当前页，可以继续管理）
loadMyPosts();
```

---

## 三个列表页的对比

| 页面 | 数据来源 | 认证 | 显示内容 | 操作按钮 |
|------|---------|------|---------|---------|
| 首页 `index.html` | `GET /posts` | 否 | 标题+摘要+作者+时间 | 无 |
| 文章详情 `detail.html` | `GET /posts/:id` | 否 | 标题+完整内容+作者+时间 | 编辑+删除（仅作者可见） |
| 我的文章 `my.html` | `GET /posts/my` | 是 | 标题+时间 | 编辑+删除（都是本人的） |

---

## 总结

- **详情页**的核心逻辑：身份判断（isOwner）→ 条件渲染操作按钮
- **我的文章**的核心逻辑：认证守卫 + 用户维度数据过滤 + 原地刷新
- 所有页面都依赖 `api.js`（自动带 Token）+ `auth.js`（状态读取）
