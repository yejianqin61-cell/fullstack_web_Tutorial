## 开场

这两个页面的结构非常相似——都是一个居中卡片 + 表单。我们重点看它们的 JS 交互逻辑：表单提交、前端校验、API 调用、状态保存和路由跳转。

## 一、注册页 [register.html]

### HTML 结构关键点

```html
<form id="registerForm">
  <div class="form-group">
    <label for="username">用户名</label>
    <input type="text" id="username" placeholder="2-20个字符" required
           minlength="2" maxlength="20" autocomplete="username">
  </div>
  <div class="form-group">
    <label for="password">密码</label>
    <input type="password" id="password" placeholder="4-50个字符" required
           minlength="4" maxlength="50" autocomplete="new-password">
  </div>
  <div class="form-group">
    <label for="confirmPassword">确认密码</label>
    <input type="password" id="confirmPassword" placeholder="请再次输入密码" required
           autocomplete="new-password">
  </div>
  <button type="submit" class="btn btn-primary btn-block">注 册</button>
</form>
<div class="auth-switch">
  已有账号？<a href="/login.html">立即登录 →</a>
</div>
```

### HTML5 表单校验属性

- **`required`** — 必填，浏览器会阻止提交空字段
- **`minlength="2" maxlength="20"`** — 字符数限制，浏览器会验证
- **`autocomplete`** — 告诉浏览器是否自动填充。`"new-password"` 表示这是新密码创建，浏览器不会用之前保存的密码自动填充

> **前端校验 vs 后端校验**：HTML5 校验只是体验优化（即时反馈），后端还有一套同样的校验逻辑——因为攻击者可以绕过浏览器直接发 HTTP 请求。

### 注册表单 JS

```javascript
document.getElementById('registerForm').addEventListener('submit', async (e) => {
  e.preventDefault();        // ← 阻止表单的默认提交行为
  hideAlert('alertBox');     // ← 先清除之前的提示
```

#### `e.preventDefault()` — 阻止默认行为

HTML `<form>` 的默认提交行为是：收集表单数据 → 拼接成 URL 参数 → 刷新页面跳转。我们用的是 AJAX 方式（fetch），不需要页面刷新，所以必须 `preventDefault()` 阻止默认行为。

```javascript
  const username = document.getElementById('username').value.trim();
  const password = document.getElementById('password').value;
  const confirmPassword = document.getElementById('confirmPassword').value;
```

#### `.value` — 获取表单控件的当前值

#### `.trim()` — 去掉首尾空格

用户名 `" alice "` → `"alice"`。密码不用 `trim()`——空格也是密码的一部分。

---

```javascript
  if (password !== confirmPassword) {
    showAlert('alertBox', '两次输入的密码不一致', 'error');
    return;
  }
```

**确认密码**的前端校验。这是纯前端逻辑——后端不关心确认密码，因为只有一份密码传过去。

---

```javascript
  try {
    await post('/auth/register', { username, password });
    showAlert('alertBox', '注册成功！正在跳转到登录页...', 'success');
    setTimeout(() => { window.location.href = '/login.html'; }, 800);
  } catch (err) {
    showAlert('alertBox', err.message, 'error');
  }
```

#### 注册成功后的流程

```
post('/auth/register', ...) → await 等待完成
  ↓ 成功
showAlert('注册成功', 'success')
  ↓
setTimeout(..., 800) → 等待 800 毫秒 → window.location.href = '/login.html'
```

- **`setTimeout` 的作用**：让用户看到"注册成功"的提示，而不是瞬间跳转
- **为什么跳转到登录页而不是直接登录？** 注册不自动签发 Token——需要用户手动登录获取 Token。这是更安全的设计

## 二、登录页 [login.html]

### HTML 结构

```html
<form id="loginForm">
  <div class="form-group">
    <label for="username">用户名</label>
    <input type="text" id="username" placeholder="请输入用户名" required autocomplete="username">
  </div>
  <div class="form-group">
    <label for="password">密码</label>
    <input type="password" id="password" placeholder="请输入密码" required autocomplete="current-password">
  </div>
  <button type="submit" class="btn btn-primary btn-block">登 录</button>
</form>
<div class="auth-switch">
  还没有账号？<a href="/register.html">立即注册 →</a>
</div>
```

- **`autocomplete="current-password"`** — 告诉浏览器这是登录用的密码，可以自动填充
- 注册和登录的底部都有"切换到另一个页面"的链接——这是 UX 的基本要求

---

### 登录表单 JS

```javascript
document.getElementById('loginForm').addEventListener('submit', async (e) => {
  e.preventDefault();
  hideAlert('alertBox');

  const username = document.getElementById('username').value.trim();
  const password = document.getElementById('password').value;

  try {
    const data = await post('/auth/login', { username, password });
    saveAuth(data.token, data.user);       // ← 关键：保存登录状态
    showAlert('alertBox', '登录成功，正在跳转...', 'success');
    setTimeout(() => { window.location.href = '/'; }, 500);
  } catch (err) {
    showAlert('alertBox', err.message, 'error');
  }
});
```

#### `saveAuth(data.token, data.user)` — 状态持久化

```javascript
// auth.js 中的实现
function saveAuth(token, user) {
  localStorage.setItem('token', token);
  localStorage.setItem('user', JSON.stringify(user));
}
```

登录成功后，token 和用户信息写入 `localStorage`。从此以后：
- `api.js` 的 `request()` 会自动从 `localStorage` 读取 token 附加到请求头
- `updateNavbar()` 会从 `localStorage` 读取 user 信息展示用户名

## 登录 vs 注册流程对比

```
注册流程：                      登录流程：
  fill form                      fill form
    ↓                              ↓
  submit                          submit
    ↓                              ↓
  前端校验（密码一致性）            post('/auth/login')
    ↓                              ↓
  post('/auth/register')          saveAuth(token, user)  ← 保存状态
    ↓                              ↓
  后端校验 + bcrypt hash          跳转首页（已登录态）
    ↓
  跳转登录页（未登录态）
```

关键区别：
- **注册**：成功后不保存登录状态，需要去登录页
- **登录**：成功后 `saveAuth()` 保存状态，后续所有请求自动带 Token
