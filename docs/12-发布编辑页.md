# 12 - create.html：发布与编辑文章

> 视频讲解稿 · 对应文件：[frontend/create.html](../../frontend/create.html)

---

## 开场

这个页面很巧妙——它**一个页面兼容两种模式**：发布新文章和编辑已有文章。通过 URL 参数 `?id=` 来区分：有 id 就是编辑模式，没有就是发布模式。

---

## 页面结构

```html
<div class="container">
  <div class="page-header">
    <h1 id="pageTitle">✏️ 写文章</h1>    <!-- 动态切换"写文章"/"编辑文章" -->
  </div>
  <form id="postForm">
    <div class="form-group">
      <label for="title">标题</label>
      <input type="text" id="title" required>
    </div>
    <div class="form-group">
      <label for="content">内容</label>
      <textarea id="content" required></textarea>
    </div>
    <button type="submit" class="btn btn-primary" id="submitBtn">发 布</button>
    <!-- 动态切换"发布"/"更新" -->
    <a href="/" class="btn btn-secondary">取 消</a>
  </form>
</div>
```

## JS 逻辑

### 1. 登录守卫

```javascript
if (!isLoggedIn()) {
  window.location.href = '/login.html';
}
```

这个代码写在 `<script>` 顶层（不在任何函数里），页面加载时立即执行。如果未登录，直接跳转到登录页——用户连表单都看不到。

> **这种守卫是前端路由保护的最简实现**，没有框架也能做。

---

### 2. 模式检测

```javascript
const params = new URLSearchParams(window.location.search);
const editId = params.get('id');
let isEditMode = !!editId;
```

#### `URLSearchParams` — URL 查询参数解析

```javascript
// 访问 /create.html?id=3
window.location.search   // → "?id=3"
params.get('id')         // → "3"

// 访问 /create.html（没有参数）
window.location.search   // → ""
params.get('id')         // → null
```

- **`!!editId`** — 将值转为布尔。`"3"` → `true`（编辑模式），`null` → `false`（发布模式）

---

### 3. 编辑模式加载文章

```javascript
async function loadPost() {
  if (!editId) return;  // 发布模式，不需要加载，直接返回

  try {
    const post = await get(`/posts/${editId}`);

    // 权限检查：当前用户必须是文章作者
    const user = getCurrentUser();
    if (user && user.id !== post.user_id) {
      alert('无权编辑此文章');
      window.location.href = '/';
      return;
    }

    // 填充表单
    document.getElementById('title').value = post.title;
    document.getElementById('content').value = post.content;

    // 切换 UI
    document.getElementById('pageTitle').textContent = '✏️ 编辑文章';
    document.getElementById('submitBtn').textContent = '更 新';
  } catch (err) {
    showAlert('alertBox', err.message, 'error');
  }
}
```

#### `.value` — 不仅可以读取，还可以设置！

```javascript
document.getElementById('title').value = post.title;
```

表单控件的 `.value` 属性是**可读可写**的——设置它等价于用户手动输入。

#### UI 动态切换

```
发布模式：              编辑模式：
┌──────────────┐       ┌──────────────┐
│ ✏️ 写文章    │  →   │ ✏️ 编辑文章   │
│              │       │              │
│ [标题输入框]  │       │ [已有标题]    │
│ [内容输入框]  │       │ [已有内容]    │
│              │       │              │
│ [发 布] [取消]│       │ [更 新] [取消] │
└──────────────┘       └──────────────┘
```

同一份 HTML，同一个表单，只是文字和内容不同。

---

### 4. 表单提交

```javascript
document.getElementById('postForm').addEventListener('submit', async (e) => {
  e.preventDefault();
  hideAlert('alertBox');

  const title = document.getElementById('title').value.trim();
  const content = document.getElementById('content').value.trim();

  try {
    if (isEditMode) {
      await put(`/posts/${editId}`, { title, content });        // = 编辑
      showAlert('alertBox', '更新成功！正在跳转...', 'success');
      setTimeout(() => { window.location.href = `/detail.html?id=${editId}`; }, 500);
    } else {
      const data = await post('/posts', { title, content });     // 发布
      showAlert('alertBox', '发布成功！正在跳转...', 'success');
      setTimeout(() => { window.location.href = `/detail.html?id=${data.postId}`; }, 500);
    }
  } catch (err) {
    showAlert('alertBox', err.message, 'error');
  }
});
```

#### 两条路径，两种 HTTP 方法

```
isEditMode === true  →  PUT  /api/posts/:id    →  更新已有文章
isEditMode === false →  POST /api/posts         →  创建新文章
```

#### 跳转目标的差异

- **发布成功** → 跳转到新文章的详情页。`postId` 来自后端 `INSERT` 返回的 `lastInsertRowid`
- **编辑成功** → 跳转到当前文章的详情页。`editId` 本身就是文章 ID

---

### 5. 初始化

```javascript
document.addEventListener('DOMContentLoaded', loadPost);
```

页面加载完成后调用 `loadPost()`。如果是发布模式，`loadPost` 直接 `return`；如果是编辑模式，加载并填充文章数据。

## 完整流程时序图

### 发布模式
```
用户打开 /create.html
  → isLoggedIn() ✓
  → editId = null → isEditMode = false
  → loadPost() 直接 return
  → 显示空白表单
  → 用户填写 → 提交
  → POST /api/posts { title, content } + Token
  → 后端: INSERT INTO posts → postId=5
  → 跳转 /detail.html?id=5
```

### 编辑模式
```
用户打开 /create.html?id=3
  → isLoggedIn() ✓
  → editId = "3" → isEditMode = true
  → loadPost() → GET /api/posts/3
  → 验证作者身份（req.user.id === post.user_id）
  → 填充表单
  → UI 切换为"编辑文章"/"更新"
  → 用户修改 → 提交
  → PUT /api/posts/3 { title, content } + Token
  → 后端: UPDATE posts SET ... WHERE id=3
  → 跳转 /detail.html?id=3
```

## 总结

这个页面展示了几个重要模式：

1. **一页两用** — 通过 URL 参数切换模式，避免创建两个几乎相同的页面
2. **前端路由守卫** — 未登录时重定向，保护敏感页面
3. **双重权限校验** — 前端检查 `isLoggedIn()` + 编辑时检查作者身份；后端还有 `authMiddleware` + `post.user_id` 校验，双重保障
