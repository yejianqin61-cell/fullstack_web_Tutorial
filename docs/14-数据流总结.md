## 场景一：用户注册

```
浏览器                                服务器                        SQLite
──────                               ──────                       ──────

1. 用户在 register.html 填写表单
   username: "alice"
   password: "mypass"
   confirmPassword: "mypass"

2. 点击「注册」
   → form submit 事件
   → e.preventDefault()          阻止默认表单提交
   → 前端校验: password === confirmPassword ✓

3. post('/auth/register',          → api.js: request()
   {username, password})            ├─ 构造 headers { Content-Type: application/json }
                                    │  (注册不需要 token)
                                    ├─ fetch POST /api/auth/register
                                    │  body: '{"username":"alice","password":"mypass"}'
                                    │
                                    ├─ Express 收到请求
                                    │  ├─ cors() → 加跨域头
                                    │  ├─ express.json() → req.body = {username, password}
                                    │  ├─ 路由匹配: POST /api/auth/register
                                    │  └─ routes/auth.js
                                    │       ├─ 校验: username 2-20字符 ✓
                                    │       ├─ 校验: password 4-50字符 ✓
                                    │       ├─ bcrypt.hashSync("mypass", 10)
                                    │       │    → "$2a$10$N9qo8uLOickgx2ZMRZoMye..."
                                    │       │    (每次运行结果不同！)
                                    │       └─ db.prepare('INSERT INTO users...')
                                    │            .run('alice', '$2a$10$N9q...')  ───→  INSERT
                                    │                                                  ↓
                                    │                                            users 表新增一行
                                    │                                            id=1, username='alice'
                                    │                                            password='$2a$10$N9q...'
                                    │                                                  ↓
                                    │       ← result.lastInsertRowid = 1  ← ← ← ← ← ┘
                                    │
   ← 200 { message: "注册成功",      ← res.json({ message: '注册成功', userId: 1 })
            userId: 1 }

4. showAlert('注册成功', 'success')
   setTimeout 800ms 后 → window.location.href = '/login.html'
   (注意: 注册成功不自动登录)
```

## 场景二：用户登录

```
1. 用户在 login.html 填写表单
   username: "alice", password: "mypass"

2. 点击「登录」
   → post('/auth/login', {username, password})

3. 服务器收到 POST /api/auth/login           db.prepare('SELECT * FROM users
                                             WHERE username = ?').get('alice')
                                                            ↓
                                             users 表查询 → {id:1, username:'alice',
                                                              password:'$2a$10$N9q...'}
                                                            ↓
                                             bcrypt.compareSync("mypass", "$2a$10$N9q...")
                                               ├─ 从哈希中取出盐值 ($2a$10$...)
                                               ├─ 用相同盐值和轮数哈希 "mypass"
                                               └─ 比较两个哈希值 → true ✓
                                                            ↓
                                             jwt.sign({id:1, username:'alice'},
                                                      'blog_secret_key_2024',
                                                      {expiresIn:'7d'})
                                               → "eyJhbGciOi...payload...signature"
                                                            ↓
   ← 200 { token, user: {id:1, username:'alice'} }

4. saveAuth(token, user)
   localStorage.setItem('token', 'eyJhbGciOi...')
   localStorage.setItem('user', '{"id":1,"username":"alice"}')

5. 跳转首页 → updateNavbar() 读取 localStorage
   → 导航栏显示 "👤 alice [写文章] [我的文章] [退出]"
```

**关键转折点**：`saveAuth()` 之后，整个前端的行为变了——导航栏显示用户名、api.js 自动带 Token、create.html 不再跳转登录页。

## 场景三：发布文章（完整链路）

这是最复杂的一条链路，涉及所有层次。

```
浏览器                                    服务器                        SQLite
──────                                   ──────                       ──────

1. 用户点击「✏️ 写文章」
   → /create.html
   → top-level JS: isLoggedIn() → localStorage.getItem('token') 存在 → 放行

2. 填写表单
   title: "我的第一篇博客"
   content: "Hello World！这是我的第一篇博客文章。"

3. 点击「发布」
   → post('/posts', {title, content})

4. api.js: request('/posts', {method:'POST', body: JSON.stringify({title, content})})
   ├─ const token = localStorage.getItem('token')
   │      → "eyJhbGciOi..."
   ├─ headers = {
   │      'Content-Type': 'application/json',
   │      'Authorization': 'Bearer eyJhbGciOi...'     ← 自动附加！
   │   }
   └─ fetch('http://localhost:8080/api/posts', {      POST /api/posts
        method: 'POST',
        headers: { ... },
        body: '{"title":"我的第...","content":"Hello World..."}'
      })
      │
      ├─ Express 收到请求                            
      │  ├─ cors() → res.setHeader('Access-Control-Allow-Origin', '*')
      │  ├─ express.json() → req.body = {title: "我的第...", content: "Hello World..."}
      │  └─ 路由匹配: POST /api/posts
      │      ↓
      │      authMiddleware(req, res, next)             ← 认证中间件！
      │       ├─ req.headers.authorization = 'Bearer eyJhbGciOi...'
      │       ├─ 提取 token = 'eyJhbGciOi...'
      │       ├─ jwt.verify(token, 'blog_secret_key_2024')
      │       │    ├─ 签名验证通过 ✓ (没有被篡改)
      │       │    ├─ exp 检查通过 ✓ (未过期)
      │       │    └─ 返回 {id:1, username:'alice', iat:..., exp:...}
      │       ├─ req.user = {id:1, username:'alice'}   ← 注入用户信息
      │       └─ next()                                 ← 放行
      │      ↓
      │      进入 routes/posts.js 的处理函数
      │       ├─ 校验: title.trim() !== '' ✓
      │       ├─ 校验: content.trim() !== '' ✓
      │       └─ db.prepare('INSERT INTO posts            ──→  INSERT
      │                      (title, content, user_id)          posts 表新增:
      │                      VALUES (?, ?, ?)')                  title='我的第一篇博客'
      │           .run('我的第一篇博客',                         content='Hello World...'
      │                'Hello World...',                         user_id=1 (来自 req.user.id!)
      │                req.user.id)  ← ← ← ← ← ← ← ← ←         ↓
      │       ← result.lastInsertRowid = 1               ← ← ← ┘
      │
   ← 201 { message: "发布成功", postId: 1 }

5. showAlert('发布成功', 'success')
   → setTimeout 500ms → window.location.href = '/detail.html?id=1'
```

## 场景四：首页加载文章列表

```
1. 用户访问首页 /
   → Express: express.static → 返回 frontend/index.html

2. 浏览器解析 HTML
   → 加载 css/style.css
   → 加载 js/api.js, js/auth.js, js/main.js
   → DOMContentLoaded 触发
      ├─ main.js: updateNavbar() → 根据 localStorage 渲染导航栏
      └─ index.html inline: loadPosts()

3. loadPosts()
   → get('/posts')
   → api.js: request('/posts', {method:'GET'})
      ├─ token = localStorage.getItem('token')
      │    → 如果有 Token 就带上（首页虽然是公开接口，但带了也没关系）
      └─ fetch('http://localhost:8080/api/posts')

4. 服务器 GET /api/posts (公开，无 authMiddleware)
   → routes/posts.js
   → SQL:
      SELECT p.id, p.title,
             substr(p.content, 1, 150) AS summary,    ← 只取前150字
             p.user_id, u.username AS author,
             p.created_at, p.updated_at
      FROM posts p
      JOIN users u ON p.user_id = u.id                ← 联表查用户名
      ORDER BY p.created_at DESC                       ← 最新的在前

5. 数据库返回:
   [{id:1, title:'我的第一篇博客', summary:'Hello World！这是...',
     user_id:1, author:'alice', created_at:'2026-06-04 11:49:53'}]

6. 服务器 → res.json(posts) → 浏览器

7. loadPosts(): posts.map(p → HTML 模板)
   → container.innerHTML = '<ul class="post-list">...</ul>'
   → 浏览器渲染文章卡片
```

## 安全体系回顾

```
┌──────────────────────────────────────────────────┐
│                  安全防护层                        │
│                                                  │
│  1. 密码安全                                      │
│     bcrypt.hashSync(password, 10)                 │
│     明文密码永不到达数据库                          │
│     即使数据库泄露，攻击者无法反推原文                │
│                                                  │
│  2. 认证安全                                      │
│     JWT: 无状态 + 签名防篡改 + 7天过期              │
│     authMiddleware: 每个受保护请求验证身份           │
│                                                  │
│  3. 授权安全                                      │
│     post.user_id !== req.user.id → 403           │
│     编辑/删除前验证操作者就是作者                    │
│                                                  │
│  4. 注入防御                                      │
│     db.prepare(SQL).run(params) ← 参数化查询        │
│     恶意输入不会被当作 SQL 执行                     │
│                                                  │
│  5. XSS 防御                                      │
│     escapeHtml() ← 所有用户数据渲染前转义            │
│     恶意脚本不会被浏览器执行                        │
│                                                  │
│  6. 错误信息                                       │
│     "用户名或密码错误" ← 不区分具体原因               │
│     防止用户枚举攻击                               │
└──────────────────────────────────────────────────┘
```

## 总结：你学到了什么

通过这个博客项目，你接触到了全栈 Web 开发的完整知识体系：

| 层次 | 知识点 |
|------|--------|
| **运行时** | Node.js module 系统 (require/module.exports), process.env |
| **Web 框架** | Express 中间件栈, Router 模块化, req/res 对象, HTTP 状态码 |
| **数据库** | SQLite, better-sqlite3 同步 API, 参数化查询, WAL 模式, 外键约束 |
| **SQL** | CREATE TABLE, SELECT/JOIN, INSERT, UPDATE, DELETE, WHERE, ORDER BY |
| **认证** | bcrypt 密码哈希, JWT 签发与验证, Bearer Token, 中间件模式 |
| **安全** | SQL 注入防御, XSS 防御, 密码加密, 权限校验, 错误信息脱敏 |
| **前端** | fetch API, localStorage, DOM 操作, 事件监听, innerHTML 模板, URLSearchParams |
| **CSS** | Flexbox 布局, 响应式 @media, box-shadow, 组件化 CSS, sticky 定位 |
| **架构** | 前后端分离, RESTful API 设计, 无状态认证, 单例模式 |

这个项目只有 16 个文件和 400 行代码，但它是一个完整的、可运行的全栈应用。理解了这个项目的每一行代码，你就掌握了 Web 开发的核心基础。
